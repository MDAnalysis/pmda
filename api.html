API for the Parallel Analysis¶

pmda implements a simple map-reduce scheme for parallel trajectory analysis [Khoslessan2017] for MDAnalysis [Gowers2016] [Michaud-Agrawal2011]. Building new parallel analysis classes is easy with the pmda.parallel.ParallelAnalysisBase base class if you can formulate the problem as a calculation over one AtomGroup for a single frame.

class pmda.parallel.ParallelAnalysisBase(universe, atomgroups)¶

Base class for defining parallel multi frame analysis

The class it is designed as a template for creating multiframe analyses. This class will automatically take care of setting up the trajectory reader for iterating, and it offers to show a progress meter.

dask_helper(start, stop, step, indices, top, traj)¶

helper function to actually setup dask graph

run(n_jobs=1, start=None, stop=None, step=None, get=None)¶

Perform the calculation

Parameters:
  • n_jobs (int, optional) – number of jobs to start, if -1 use number of logical cpu cores
  • start (int, optional) – start frame of analysis
  • stop (int, optional) – stop frame of analysis
  • step (int, optional) – number of frames to skip between each analysed frame
